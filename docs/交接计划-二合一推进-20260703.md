## 一、本次已落地

### 1. browser_* 「当前页」跟踪 (上一对话定位的关键缺陷 · 本 PR 修复)

- **病灶**: `daoCdpPickPage()` 无 targetId 时恒返 `pages[0]`(常是遗留 about:blank)，
  而 `browser_navigate` 新建标签后从不记录——`snapshot/get_text/get_html/click/type/fill_form`
  全部落到空白标签 → "element not found" / 空 DOM。
- **治法** (`core/dao-vsix/src/extension.ts`):
  - 模块级 `daoCdpActiveTarget` 跟踪当前页(Playwright 语义)。
  - `daoCdpPickPage`: 显式 targetId → 记为当前页；无 targetId → 优先当前页(仍存活)，否则回落首页。
  - `browser_navigate` / `browser_tabs(new|select|close)` 维护当前页。
  - 新增 `browser_activate` 工具(显式设当前页)；`browser_targets` / `browser_tabs list`
    输出携带 `active` 标记。工具总数 63 → 64。
- **验证**: `node build.js` ✓ · `node tools/render_check.js` ✓ · rt-flow `npm test` 35/35 ✓ ·
  vendor diff ✓ (rt-flow 未改动)。live 验证待本地桥接令牌恢复(见 三)。
